# checklistスキル

提出前の最終チェックを行う。

## コマンド

```text
/checklist
```

---

## 手順

### Step 1: チェックリストを表示する

```markdown
# 提出前チェック

- [ ] 対象はウェブアプリケーションである
- [ ] URLを記入した
- [ ] 確認日を記入した
- [ ] 再現手順を書いた
- [ ] 発見した内容を書いた
- [ ] 脆弱性の種類を選んだ
- [ ] CWE候補を書いた
- [ ] 機密性・完全性・可用性の影響を書いた
- [ ] 検証コードの有無を整理した
- [ ] スクリーンショットの有無を整理した
- [ ] ログの有無を整理した
- [ ] すでに他所へ連絡した場合は記入した
- [ ] 不要な公開・拡散をしていない
- [ ] 断定しすぎる表現を避けた
```

### Step 2: 足りない項目だけ指摘する

全部を言い直さない。
未記入・弱い箇所だけ挙げる。

例:

```text
- 再現手順はあるが、最終的に何が見えたかが弱い
- CWE候補はあるが、第一候補だけに絞った方がよい
- ログファイルの有無が未記入
```

### Step 3: checklist.md に保存する


```text
.ipa-sec-todokede/checklist.md
```
